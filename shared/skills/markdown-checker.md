# skills/markdown-checker.md
# Markdown品質チェックスキル
# すべてのプロジェクトで使用する共通スキル

---

## このスキルの役割

生成されたMarkdownファイルの品質を検証する。
著者の人格チェック・Markdown記法チェック・誤字チェックを行う。

---

## チェック項目

### 1. Markdown記法チェック

#### 太字・強調
- `**テキスト**` の閉じタグが欠けていないか
- `*テキスト*` の閉じタグが欠けていないか
- 太字の中に不要なスペースがないか（`** テキスト **` は誤り）

#### 見出し
- `#` の後にスペースがあるか（`#見出し` は誤り、`# 見出し` が正しい）
- 見出しレベルが飛んでいないか（h1→h3 のように h2 を飛ばさない）
- 見出しの後に空行があるか

#### リスト
- `- ` または `* ` の後にスペースがあるか
- ネストのインデントが統一されているか（スペース2つまたは4つで統一）

#### コードブロック
- バッククォート3つで開始・終了しているか（`` ``` ``）
- 言語指定があるか（`` ```cpp ``、`` ```markdown `` など）
- コードブロックの前後に空行があるか

#### テーブル
- ヘッダー行と区切り行（`|:---|:---|`）があるか
- 列数がすべての行で一致しているか

#### リンク・画像
- `[テキスト](URL)` の形式が正しいか
- 閉じ括弧が欠けていないか

---

### 2. 誤字・メモ残留チェック

#### 削除し忘れのメモ
以下のパターンが残っていないか確認する：
- `TODO:` `FIXME:` `XXX:` `NOTE:` `MEMO:`
- `【ここに〜を書く】` `[ここに〜]` `〜を追加予定`
- `（仮）` `（未記入）` `（後で確認）`
- `<!--` と `-->` で囲まれた執筆指示コメントが出力に残っていないか

#### 明らかな誤字パターン
- 同じ語句の直後重複（「の の」「は は」など）
- 句読点の連続（「。。」「、、」）
- 閉じ括弧の欠け（「（テキスト」で終わっている）

---

### 3. 著者の人格チェック

shared/skills/author-voice.md の「やらないこと」チェックリストを適用する。

- 「〜すべき」「〜べきです」の断定がないか
- 否定的なラベルがないか（スパゲッティ・ダメ・最悪・地獄）
- 著者の口癖が3〜5箇所散りばめられているか

---

## 出力形式

```json
{
  "status": "ok|issues-found",
  "issues": [
    {
      "type": "markdown|typo|author-voice",
      "line": 42,
      "description": "具体的な問題の説明",
      "original": "問題のある元のテキスト",
      "suggestion": "修正案"
    }
  ]
}
```
