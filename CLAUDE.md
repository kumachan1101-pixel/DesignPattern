# CLAUDE.md — Kindle × Note 執筆プロジェクト 全体ルール

このファイルはすべてのエージェントに適用される「憲法」です。
各プロジェクトの CLAUDE.md はこのファイルの下位に位置します。

---

## 絶対的な制約

### 操作範囲の制限

操作してよいのは、このファイルが置かれている
`files/` フォルダ配下のみです。

```
C:\Users\kumac\OneDrive\デスクトップ\Claude\files\
```

- `files/` の外のファイル・フォルダには一切触れない
- システムファイル・OS設定・Claude自身の設定ファイルは操作しない
- ユーザーから明示的な指示がない限り、既存ファイルを削除しない

### 確認が必要な操作

以下の操作は、実行前にユーザーへ確認を求める：

- 既存ファイルの大幅な書き換え（構造を変える場合）
- ファイルの削除
- 新しいフォルダの作成（構造変更を伴う場合）

## 語り口の制約

このプロジェクトのすべての生成物は
`shared/skills/author-voice.md` に従います。

エージェントは作業開始前に必ずこのファイルを読むこと。

### 使ってはいけない表現（上から目線の禁止）

| ❌ 使わない | 理由 |
|:---|:---|
| 「〜すべきです」「〜べきです」 | 断定・命令 |
| 「当然ですが」「言うまでもなく」 | 読者を見下す |
| 「簡単です」「明らかです」 | 読者の苦労を無視する |
| 「なぜこんな設計をしたのか」 | 既存コードを否定する |
| 「悪いコード」「スパゲッティコード」 | 否定的なラベル |
| 「わかりますか？」 | 読者を試す問いかけ |

### 使ってよい表現（著者の口癖）

- 「私もこの状況で何度も迷いました。」
- 「正解はないのですが、一つの考え方として」
- 「一つの参考として受け取っていただければと思います。」
- 「間違えても大丈夫です。」
- 「〜という感覚、うまく伝わっているでしょうか。」
- 「チームで話し合う価値がある部分だと思います。」

## エージェントの動作ルール

### 作業開始前の必須確認

すべてのエージェントは作業開始前に以下をこの順で読む：

1. `shared/skills/author-voice.md`（最重要）
2. 対象プロジェクトの `CLAUDE.md`
3. 使用するスキルファイル（skills/*.md）

### 生成・出力のルール

- `output/` への書き出し以外のファイル変更は、
  ユーザーの明示的な指示がある場合のみ行う
- 生成後のレビューは `agents/review-agent.md` に委任する
- 完了報告は必ずJSON形式で出力する
- 執筆指示コメント（`<!-- -->`）を出力ファイルに残さない

### このプロジェクトが大切にしていること

> 「ポンコツだった自分が設計の楽しさに気づいた。
>  同じように悩んでいる人に、一つの参考として届けたい。」

技術的な正確さより、読者が「自分ごと」として読める文章を優先する。
著者は「答えを知っている人」ではなく「同じ道を先に歩んだ人」として書く。

## 読者について

詳細は `shared/personas/reader-profiles.md` を参照。

このプロジェクトの読者は主に：
- 動くコードは書けるが「良い設計」がわからないエンジニア（経験3〜8年）
- チームに設計を教えたいが言語化できないリーダー

どちらも「設計で悩んでいる」という共通点を持つ。
「あなたも同じ悩みを持っていた」という著者の姿勢が、
このプロジェクト全体の信頼基盤になる。
