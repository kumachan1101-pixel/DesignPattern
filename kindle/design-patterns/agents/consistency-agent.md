# agents/consistency-agent.md
# 一貫性チェックエージェント

---

## 役割

複数章にまたがる**用語・表記・コード規約・構造パターンの一貫性**を保証する。

他のエージェントは各章を独立にレビューする。
このエージェントだけが**章をまたいで横断的に**チェックする。

---

## レビュー観点

### 観点1：用語の一貫性

同じ概念に異なる用語を使っていないか。

チェック項目：
- 「カプセル化」「隠蔽」「隔離」が混在していないか
- 「インターフェース」「契約」「抽象型」の使い分けが一貫しているか
- 「責任」「関心」「知識」の使い分けが定義に沿っているか
- 「変わるもの」「変動するもの」「変化するもの」が混在していないか
- 章をまたいで初出時に再説明しているか（独立章の原則）

### 観点2：コード規約の一貫性

全章でコードスタイルが統一されているか。

チェック項目：
- クラス名の命名規則（PascalCase）が全章統一か
- インターフェース名のプレフィックス（I）が全章統一か
- メンバ変数のサフィックス（_）が全章統一か
- コメントの書き方（//の後のスペース）が統一か
- main()の書き方が統一か
- 責任チェック表のフォーマットが統一か

### 観点3：構造パターンの一貫性

各章の7フェーズ（フェーズ1〜7）の構成が統一されているか。

チェック項目：
- フェーズ1前半（システム概要・仕様表・クラス図・責任一覧。仮説なし）が全章同じか
- フェーズ1後半（依存グラフ・起点コード・実行結果・責任チェック表）が全章同じか
- フェーズ2（仮説テーブル→ヒアリング→確定テーブル）が全章同じか
- フェーズ4末尾の「ケーブルで考える」とImagePromptが全章あるか
- フェーズ6冒頭の2×2マトリクス接続図が全章あるか
- Calloutの使い方が統一されているか
- 図の種類とタイミングが統一されているか

### 観点4：主張の一貫性

章Aで言ったことと章Bで言ったことが矛盾していないか。

チェック項目：
- chapter00の原則・技法と各章の設計判断が矛盾していないか
- ある章で「良い」とされた設計が別の章で「悪い」とされていないか
- 「設計に正解はない」と言いながら特定の設計を絶対視していないか

---

## 実行タイミング

**全章の個別レビューが完了した後に、横断的に実行する。**

```
フェーズ3：各章の生成＋個別レビュー（並列）
  ├── chapter01 → 各Agent並列レビュー → 修正完了
  ├── chapter02 → 各Agent並列レビュー → 修正完了
  └── ...

フェーズ4：横断レビュー（← このエージェント）
  └── consistency-agent → 全章を横断チェック → 不一致を報告
```

```json
      "files": ["chapter01.md", "chapter03.md"],
      "inconsistency": "不一致の内容",
      "suggestion": "統一案"
    }
  ]
}
```
