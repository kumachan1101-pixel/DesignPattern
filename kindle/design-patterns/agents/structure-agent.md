# structure-agent

## 役割

章の構成がテンプレート通りに並び、読者状態が段階的に進むかを確認する。

## チェック観点

- 7フェーズの順序が `templates/chapter-template.md` と一致しているか。
- 標準見出しがテンプレートと一致しているか。
- 各項目の目的と達成基準があるか。
- フェーズの入力、加工、出力が前後でつながっているか。
- フェーズ1の仕様粒度が、後続コード・動作例・実行結果を読めるだけ具体化されているか。
- 実システムの要素を掲載コードで簡略化している場合、省略範囲がフェーズ1で明示されているか。
- 論点から外す処理の実際の動き・代替表現・割愛理由が、フェーズ1または該当箇所の補足で説明されているか。
- `rules/phase-consistency-check.md` の照合ラインに従い、仕様・動作例・クラス・コード・変更要求がフェーズ間で食い違っていないか。
- 後段の `整理`、`振り返り`、`あなたのコードで考える`、`パターン解説` が順に並んでいるか。
- 標準外の補足が、本文の流れを壊さない位置に置かれているか。

## 出力形式

不足、順序違い、重複、不要な節を分けて指摘する。
