# rules/recurrence-prevention.md
# 修正したら原因分析と再発防止を行う仕組み

章・ルール・スクリプトを修正したら、その場を直して終わりにしない。
同じ指摘が別の章や次回の作業で再発しないよう、原因を正本側へ反映する。

## 必ず回す4ステップ

1. **症状を1行で書く** — 何がどう間違っていたか（読者が困る形で）。
2. **原因を1行で書く** — なぜ起きたか。特に「正本（テンプレート／ルール／チェックリスト／Agent／検証スクリプト）の不足・矛盾・陳腐化」から来ていないかを必ず疑う。直した章だけが悪いのか、正本が再発を許しているのかを切り分ける。
3. **正本を直す** — 原因が正本側にあるなら、章だけでなく次を更新する。
   - 構造・見出し・項目の不足や矛盾 → `templates/chapter-template.md`、`output/chapter00_2.md`。
   - 書き方・粒度の基準不足 → `rules/writing-rules.md`、`rules/checklist.md`、`rules/phase-consistency-check.md`。
   - レビュー観点の漏れ → 該当 `agents/*.md`。
   - 機械的に判定できるもの → `script/validate_book.py`（必須見出し・禁止表現・構造）か `script/audit_book.py`（コンパイル・表幅・リスク表現）のチェックへ落とす。「人が毎回気をつける」で終わらせない。
4. **このファイルのログ表へ1行記録する** — 同じ原因が再発したとき、過去の対処を辿れるようにする。

## 正本どうしの結合（変えたら一緒に直す）

- テンプレートの標準見出し・フェーズ名・項目名を変えたら、`script/validate_book.py` の `REQUIRED_PHASES` / `REQUIRED_NUMBERED_SECTIONS` も同じ語へ更新する。片方だけ直すと全章が誤って失敗する。
- テンプレートに必須の図・表・節を足したら、`rules/checklist.md` に対応するチェック行を足す。
- 禁止表現を増やしたら、可能なら `script/validate_book.py` の `BANNED_PATTERNS` へ追加する。

## 機械チェックにできるか迷ったときの基準

- 見出し・語句・構造の有無 → 機械チェック向き（validate へ）。
- コンパイル可否・表の列数・特定表現の混入 → 機械チェック向き（audit / validate へ）。
- 「分かりやすいか」「粒度が仕様に紐づくか」「同じ対象を追えているか」 → 人の判断。checklist と phase-consistency-check の観点を使い、Agent レビューで担保する。
