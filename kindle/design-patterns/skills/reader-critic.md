# skills/reader-critic.md
# 読者批評スキル — 章を読者の目線で批評し、疑問を洗い出す

---

## このスキルの役割

章を「この本の読者」になりきって読み、
「ここで疑問が生まれる」「ここで混乱する」「ここで信頼が下がる」
という箇所を全て洗い出す。

著者視点ではなく読者視点で読む。
「自分が書いたから分かる」ではなく「初めて読む読者には分からない」を探す。

---

## 想定読者プロファイル

`../../shared/personas/reader-profiles.md` を参照。前提として：
- 現役のエンジニア。日々コードを書いている。
- デザインパターンを「名前は聞いたことがある」程度。
- 自分のコードに適用しようとして「使えない」と感じた経験がある。
- 設計の本を読んで「分かったような気がするが、現場では使えなかった」という体験を持つ。
- C++ に詳しくない場合がある。言語仕様より設計の話を求めている。

この読者は賢いが、説明なしに前提を共有しない。
「分かるだろう」と省略された部分で止まる。

---

## 疑問の7カテゴリ

| 記号 | カテゴリ | 代表的な問い |
|---|---|---|
| 📖 | **用語未定義** | この言葉が初めて出てきたとき、意味が分からない |
| 🔗 | **論理の飛躍** | AからBへの説明の間に何かが省略されている |
| ⚡ | **整合性の矛盾** | 前でAと言ったのに、ここではBと言っている |
| 🔨 | **実践への疑問** | 実際にどうやるのか、基準や手順が不明 |
| 🔄 | **反例・例外の疑問** | でもXの場合はどうなるの？ |
| 👁 | **省略・不足** | ここで説明が突然終わっている。先が気になる |
| 💡 | **なぜそれが重要か** | 事実は分かったが、それが自分にとってなぜ意味があるのか腹に落ちない |

---

## 深刻度の分類

| ラベル | 意味 |
|---|---|
| 🔴 ブロッキング | この疑問が解消されないと、誤った理解のまま先を読み続ける。または読むのを止める。 |
| 🟡 重要 | 理解が浅くなる。著者への信頼が下がる。「なんかモヤっとする」感覚が残る。 |
| 🟢 軽微 | 気になるが読み進められる。気づかない読者もいる。 |

---

## 実行手順

### 1. 章を通読する（読者として）

著者コメントや意図を知っている状態で読んではいけない。
「この読者は今この行を初めて読んでいる」という前提で読む。

各段落ごとに問う：
- この段落で使われている言葉は、ここより前に定義されていたか
- この段落の結論は、前の段落から論理的に導かれているか
- この段落を読んで「でも…」という反論や疑問が自然に浮かぶか
- この段落の説明で「自分の現場でこれをどうやるか」の答えが出るか

### 2. 疑問を記録する

疑問が生まれたら、以下の情報を記録する：

```
- 箇所: [章.節番号、または節のタイトル]
- 読者の言葉: 「〜ということは、〜ということ？」「なぜ〜なのか分からない」（一人称）
- カテゴリ: 📖/🔗/⚡/🔨/🔄/👁/💡
- 深刻度: 🔴/🟡/🟢
- 修正の方向性: どう変えれば疑問が消えるか（1行）
```

### 3. 整理して出力する

深刻度順（🔴 → 🟡 → 🟢）にソートして出力する。
同じ深刻度の中では、章の登場順に並べる。
各疑問は読者の一人称で書く（「〜が分からない」「〜と感じる」）。

...

## 確認観点チェックリスト（見落とし防止）

以下の観点は見落としやすい。必ず確認すること。

- [ ] 章の冒頭で使われているキーワードは、この章（または第0章）で定義されているか
- [ ] 用語が複数の名前で呼ばれていないか（例：「責任を分離する」「カプセル化する」「分ける」が同じ意味で混在）
- [ ] コードの変数名・クラス名に、説明なしに規約（I prefix = interfaceなど）が使われていないか
- [ ] ヒアリングの結果が「都合よく」設計を正当化していないか（読者が「出来レースだ」と感じないか）
- [ ] 「設計に絶対の正解はありません」のような免責フレーズが多用されていないか（繰り返すと「では何が基準か」という疑問になる）
- [ ] 前の章・後の章への依存がないか（各章は独立して読める設計のはず）
- [ ] 「自分の現場で使うとしたら、最初の一歩は何か」が分かるか
- [ ] 著者の主観的な感想（「うまく伝わっているでしょうか」「私自身も迷いました」）が多すぎないか
- [ ] 図（mermaid）の説明は図を見なくても文章だけで理解できるか。図と文章が説明を重複していないか
