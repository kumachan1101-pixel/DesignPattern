# 接続点図の仕様

旧分類マトリクス画像は廃止した。

今後、章へ接続点の図を追加する場合は、分類図ではなく次の情報を示す。

## 必須要素

- 左側：変更から守りたい処理・クラス
- 右側：変わる知識を持つ処理・クラス
- 中央：接続点
- 接続点を越える値・型・操作・イベント
- 呼び出し元へ漏れている知識
- 変更要求を当てたときの修正箇所

## 表示例

```text
[注文計算の骨格]
        |
        | 合計金額を渡す / 割引後金額を受け取る
        v
[割引ルール]

呼び出し元へ漏れている知識:
- 顧客種別の条件
- キャンペーン期間の条件
- 割引率
```

## 色

- 青：変更から守りたい責任
- 橙：変わる知識
- 赤：現在、境界を越えて漏れている知識
- 緑：対策後に残す安定した約束

## 禁止

- 固定的な4分類
- 軸と象限を持つマトリクス
- パターン名を分類セルへ割り当てること
- 図だけで良し悪しを断定すること
