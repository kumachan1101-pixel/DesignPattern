# 第2章 Facadeパターン ―― 複雑なサブシステムへの「窓口」を1つにまとめる

## ステップ0：システムを把握し、仮説を立てる

**この章で扱うシステム：月次給与処理バッチ**

給与計算、勤怠管理、PDF出力という3つの外部サービスを組み合わせて給与明細を生成するバッチシステムです。

**変動と不変の仮説**

| 分類 | 仮説 | 根拠 |
|---|---|---|
| 🔴 **変動する** | 各サービスのAPI仕様やデータ形式 | 外部システムであり、自分たちの都合で変更できないため |
| 🟢 **不変** | 「給与を計算し、勤怠を確認し、明細を出す」という業務フロー | 給与支払いの業務プロセスそのものは安定しているため |

---

## ステップ1：実装コードを読む ―― 知識が多すぎるクラス

### 2.1 現状のコード

```cpp
class MonthlyBatch {
public:
    void run() {
        payrollService.calc(empId); // 詳細な具体クラスに依存
        laborService.getWorkHours(empId); // 詳細な具体クラスに依存
        pdfService.generate(empId); // 詳細な具体クラスに依存
    }
};
```

**問題の発見：**
`MonthlyBatch` が3つの外部サービスの具体クラスとその詳細な使い方をすべて知ってしまっています。

---

## ステップ2：仮説を確定する ―― 関係者ヒアリング

**関係者の声：**
- **情シス：** 「来期から給与システムが刷新され、APIの形式が大幅に変わります。」

---

## ステップ3：課題分析 ―― 変更が来たとき、どこが辛いか

外部APIが一つ変わるだけで、メインロジックである `MonthlyBatch` を修正しなければなりません。業務フローとは関係ない「APIの事情」で大切なコードを触ることになります。

---

## ステップ4：原因分析 ―― 困難の根本にある問題を言語化する

**真因：** 呼び出し元が、背後にあるサブシステムの「実装詳細」を知りすぎていること。

---

## ステップ5：対策案の検討 ―― 方向性を決め、手段を順に試す

### 2.5.1 方向性の特定

原因が「知りすぎている」なら、解消する方向性は一つです。

**→ 依存を一つの窓口に「統合する」。**

---

### 2.5.2 手段①：窓口クラスでまとめる（直感的な案）

単に呼び出しを一箇所に集める専用クラス（コーディネーター）を作る方法です。

```cpp
class PayrollCoordinator {
public:
    void process(Id id) {
        payroll.calc(id);
        labor.check(id);
        pdf.out(id);
    }
};
```

**残る課題：**
依存の場所は一箇所に減りましたが、依然として「具体クラスへの依存」は残っています。

---

### 2.5.3 手段②：インターフェース層を導入する（発想の転換）

第0章の手札：「外部依存のAPI形式が変わる」→ **窓口集約** ＋ **インターフェース抽出**。

**「窓口とサブシステムの間に契約を導入し、背後を完全に隠蔽する。」**

このように、複雑なサブシステムに一つのシンプルな窓口を提供し、実装を隠す構造を **Facade（ファサード）パターン** と呼びます。

---

## ステップ6：天秤にかける ―― 手段①と手段②を比べる

### 2.6.1 評価軸の宣言

1. **変更の局所性**：API変更時に修正するクラスを一つに絞れるか
2. **テストの独立性**：実機がない環境でもバッチを動かせるか

### 2.6.2 手段①vs手段②の比較

| 評価軸 | 手段①（単なる集約） | 手段②（Facadeパターン） |
|---|---|---|
| 変更の局所性 | ❌ 窓口クラスの修正が必要 | ✅ 実装クラスの差し替えのみで済む |
| テストの独立性 | ❌ 外部依存が残りテストしづらい | ✅ 完全にスタブ化できる |

**結論：外部API変更が確定しているため、手段②を採用します。**

---

## ステップ7：決断と、手に入れた未来

### 2.7 解決後のコード（構造）

```cpp
class IPayrollFacade {
public:
    virtual void processPaySlip(Id id) = 0;
};

class MonthlyBatch {
    IPayrollFacade* facade;
public:
    void run() { facade->processPaySlip(empId); }
};
```
