#### 視点1：接続点の特定

ステップ4で「分ける」と判断した場所に、接続点（ジョイント）が生まれます。その接続点がどこに何個あるかを明確にします。

```
%% 接続点を列挙するプレースホルダー
%% - 接続点A：【クラス名】 ←→ 【クラス名】 の境界
%% - 接続点B：【クラス名】 ←→ 【クラス名】 の境界
```

接続点が複数ある場合、それぞれ独立に課題を定義します。

#### 視点2：非機能制約の確認

<!-- 変更頻度・パフォーマンス・メモリへの影響を確認する -->

| **確認項目** | **内容** | **この章での判断** |
|---|---|---|
| 変更頻度 | この接続点はどのくらいの頻度で変わるか | 【高 / 中 / 低】 |
| パフォーマンス | ホットパスか（高頻度で呼ばれるか） | 【はい / いいえ】 |
| メモリ | 間接層の追加でオーバーヘッドが問題になるか | 【はい / いいえ】 |

ホットパスに位置する接続点は、接続の形の選択肢が「具体×直接」寄りに絞られます。

#### 視点3：クライアントへの影響範囲

<!-- クライアント＝分離対象のクラスを呼び出している既存コード -->

```
%% 影響範囲の列挙プレースホルダー

```

クライアントへの影響が大きいほど、段階的なリファクタリングが必要になります。
