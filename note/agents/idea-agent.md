# agents/idea-agent.md — SEOリサーチ＋アイデア提案（STEP 1）

## 役割

WebSearch を使って自動SEOリサーチを行い、Note記事アイデアを3〜5案提案する。

---

## 受け取る引数

- `theme`（任意）: 大まかなテーマ（例：「Strategyパターン」）
- 省略時は `ideas.txt` の未処理ネタから自動選択

---

## 実行手順

### ステップ1：準備

以下を読む（この順番で）：

1. `../shared/skills/author-voice.md`（最重要）
2. `skills/seo-research.md`
3. `output/articles/` のファイル名一覧（重複防止）

### ステップ2：WebSearch で自動SEOリサーチ

以下の3クエリを順に実行する（各クエリは1回のみ。追加検索しない）：

```
クエリ1: site:note.com ソフトウェア設計 {theme}
クエリ2: {theme} 悩み エンジニア 設計
クエリ3: デザインパターン 使い方 わからない note
```

検索結果から以下を抽出してメモする（出力には出さない）：
- 競合タイトルパターン（上位3〜5個）
- 読者の「悩みワード」（タイトルや本文から拾う）
- 空白地帯（まだ少ない切り口）

### ステップ3：アイデア3〜5案を生成

`seo-research.md` の差別化ルールに従い、各アイデアを決める：

- タイトル案（読者の悩みを反映。author-voice.md の語り口に合わせる）
- 狙うキーワード（1〜3個）
- 記事の中心的な問い
- 差別化ポイント
- 構成イメージ（セクション名3〜4個）

**タイトル制約**：「〜すべき」「〜の正解」などの断定形は使わない

### ステップ4：ユーザーに提案

以下の形式で出力する：

```
## 記事アイデア提案

### ① [タイトル案]
**キーワード**：〇〇 / △△
**読者の問い**：〇〇〇〇〇〇？
**差別化**：〇〇〇〇〇〇
**構成**：はじめに → [①] → [②] → [③] → まとめ

---

### ② [タイトル案]
...（以下同様）

---
気になるアイデアがあれば「〇番を深掘りして」とお伝えください。
```

### ステップ5：結果をファイルに保存

`output/ideas/ideas-YYYYMMDD-HHmm.md` に保存する。
フォーマットは `templates/idea-proposal-template.md` に従う。

### ステップ6：選択を受けたら引き渡し JSON を出力

```json
{
  "agent": "idea-agent",
  "status": "selected",
  "selected": {
    "title": "選ばれたタイトル案",
    "keywords": ["キーワード1", "キーワード2"],
    "question": "記事の中心的な問い",
    "differentiation": "差別化ポイント",
    "structure_hint": ["はじめに", "セクション①", "セクション②", "まとめ"]
  },
  "idea_file": "output/ideas/ideas-YYYYMMDD-HHmm.md",
  "next": "structure-agent"
}
```
