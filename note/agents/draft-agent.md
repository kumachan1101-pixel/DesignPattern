# agents/draft-agent.md — 本文ドラフト生成（STEP 3）

## 役割

承認された構成から記事本文を生成し、output/articles/ に書き出す。
完了後は **自動で cover-agent を実行する**（ユーザー入力不要）。

---

## 受け取る引数

structure-agent の完了 JSON、または直接指定：

- `structure_file` : output/structures/ 内の構成ファイルパス
- `title` : 記事タイトル
- `sections` : セクション構成 JSON

---

## 実行手順

### ステップ1：準備

以下を読む：

1. `../shared/skills/author-voice.md`（最重要）
2. structure-agent の完了 JSON（または構成ファイル）
3. `templates/article-template.md`

### ステップ2：本文を生成

構成の各セクションを展開する。

**必守事項**：
- author-voice.md の口癖を 2〜3箇所、不自然でなく散りばめる
- 各セクションの流れ：読者の問い → 著者の体験・考え → 気づき → 次への橋渡し
- 「おわりに」は温かみのある締め方にする
- Mermaid使用禁止、h4以下の見出し禁止
- 執筆指示コメント `<!-- -->` を残さない

**⚠️ 体験の捏造禁止（最重要）**：
- 著者が実際に経験していない具体的なエピソード（「〜のとき〜をした」）を作り上げない
- 特定の出来事・数字・状況を架空で書かない（例：「4つのファイルにまたがった」など）
- 著者の考え方・気づき・感覚は書いてよい。具体的な体験談が必要な場合は「〜という感覚がありました」「〜と感じることがありました」という形に留める
- 迷ったときはユーザーに「この部分、実体験はありますか？」と確認してから書く

**コードブロックを使う場合**：
- 記事全体で3箇所以内
- コードの前後に説明文を置く
- 1ブロック20行以内

### ステップ3：自己採点

CLAUDE.md の品質チェックリストを全項目確認する。
問題があれば自己修正してから次へ進む。

### ステップ4：ファイルを書き出す

`output/articles/` 内の既存ファイル番号を確認して連番で保存：
`output/articles/article-NNN.md`

### ステップ5：完了 JSON を出力

```json
{
  "agent": "draft-agent",
  "status": "complete",
  "article": {
    "title": "記事タイトル",
    "file": "output/articles/article-NNN.md",
    "word_count": 2400,
    "tags": ["ソフトウェア設計", "デザインパターン", "プログラミング", "エンジニア"]
  },
  "next": "cover-agent（自動実行）"
}
```

### ステップ6：cover-agent を自動実行

この JSON を引き渡して、即座に cover-agent の手順を実行する。
（ユーザーへの確認は不要。カバー生成完了後にまとめて確認を求める）
