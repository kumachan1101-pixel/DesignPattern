# skills/seo-research.md — SEOキーワードリサーチ指針

## このプロジェクトのSEO方針

Note.com はプラットフォーム内の「おすすめ」「フォロワー」経由が主。
検索エンジン最適化より **読者の共感・発見** を優先する。
タイトルには「読者が実際に使う言葉」を入れ、キーワード詰め込みはしない。

---

## 調査対象：読者の「悩みワード」

ペルソナA（開発経験3〜8年）が検索に使う言葉：

```
設計 わからない / リファクタリング どこから / デザインパターン 使えない
設計 学び方 / 依存関係 複雑 / クラス 分け方
責任 どこ / 変更に強い 設計 / テスト しにくい
```

---

## WebSearch クエリテンプレート

以下の3クエリを1回ずつ実行する（追加検索しない）：

```
クエリ1: site:note.com ソフトウェア設計 {theme}
クエリ2: {theme} 悩み エンジニア 設計
クエリ3: デザインパターン 使い方 わからない note
```

---

## 差別化の切り口

| 競合が多い角度 | 差別化できる角度 |
|:---|:---|
| パターンの「説明・解説」 | 「なぜ使うか」の思考プロセス |
| 理想の設計の「完成形」 | 「どう悩んで、どう判断したか」の過程 |
| 「正解」の提示 | 「チームで考えた」過程の共有 |
| 技術的に正確な解説 | 「初めて気づいた瞬間」の感覚の言語化 |

---

## キーワード選定の優先順位

1. 読者が「自分ごと」として使う言葉（共感ワード）
2. 検索ボリュームが存在する一般的な技術用語
3. タイトルに自然に収まる長さ（20文字以内）
